Enum Naming Convention

#janice_attribute, janice_color_space, janice_error and enum values follow a CAPITAL_CASE naming convention. Functions #janice_attribute_to_string and janice_error_to_string return a string for the corresponding enum by removing the leading janice_:

janice_attribute_to_string(janice_RIGHT_EYE_X); // returns "RIGHT_EYE_X"

Functions #janice_attribute_from_string and janice_error_from_string provide the opposite functionality:

janice_attribute_from_string("RIGHT_EYE_X"); // returns janice_RIGHT_EYE_X
Note
#janice_attribute_from_string is used to decode janice_metadata files, so attribute column names should follow this naming convention.
