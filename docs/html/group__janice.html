Detailed Description

Mandatory interface for the JanICE API.

Overview

A JanICE application begins with a call to janice_initialize. New templates are constructed with janice_allocate_template and provided image data with janice_detect followed by janice_augment. Templates are finalized prior to comparison with janice_finalize_template, and freed after finalization with janice_free_template. Templates can be stored on and loaded from disk using the janice_write_template and janice_read_template respectively.

Finalized templates can be used for verification with janice_verify, or search with janice_search. Galleries are managed with janice_allocate_gallery, janice_enroll, janice_remove_template and janice_free_gallery.

A JanICE application ends with a call to janice_finalize.

Thread Safety

All functions are marked one of:

  • thread-safe
    Can be called simultaneously from multiple threads, even when the invocations use shared data.
  • reentrant
    Can be called simultaneously from multiple threads, but only if each invocation uses its own data.
  • thread-unsafe
    Can not be called simultaneously from multiple threads.

Implementer Notes

Define JANICE_LIBRARY during compilation to export JanICE symbols.

typedef std::pair<int, double> cluster_pair

Cluster a collection of unlabelled people into distinct identites.

Bounding boxes for all subjects of interest in the media will be provided.

Clusters

Clusters are represented as a list of lists of <int, double> pairs. Each pairing consists of the cluster id and the cluster confidence and corresponds to an instance of an identity in a single piece of janice_media.

Clustering Hint

Clustering is generally considered to be an ill-defined problem, and most algorithms require some help determining the appropriate number of clusters. The hint parameter helps influence the number of clusters, though the implementation is free to ignore it. The goal of the hint is to provide an order of magnitude estimation for the number of identities that appear in a set of media. As such it will be a multiple of 10 (10, 100, 1000 etc.).

Note
The implementation of this function is optional, and may return JANICE_NOT_IMPLEMENTED.
Parameters
[in]associationsThe collection of media and relevant detection information to cluster
[in]hintA hint to the clustering algorithm, see Clustering Hint.
[out]clustersA list of lists of cluster pairs, see Clusters.
Remarks
This function is thread_safe.

typedef struct janice_media janice_media

Common representation for still images and videos.

Pixels are stored in row-major order. In other words, pixel layout with respect to decreasing memory spatial locality is channel, column, row frames. Thus pixel intensity can be retrieved as follows:

1 janice_data get_intensity(janice_media media, size_t channel, size_t column,
2  size_t row, size_t frame)
3 {
4  const size_t columnStep = (image.color_space == JANICE_BGR24 ? 3 : 1);
5  const size_t rowStep = image.width * columnStep;
6  const size_t index = row * rowStep + column * columnStep + channel;
7  return image.data[frame][index];
8 }

Coordinate (0, 0) corresponds to the top-left corner of the image. Coordinate (width-1, height-1) corresponds to the bottom-right corner of the image.

Build a list of templates from a single piece of janice_media.

There is no guarantee on the number of individual subjects appearing in the media and implementors should create as many templates as necessary to represent all of the people they find.

Implementors must return a single janice_track for each template they create. The track must have all of the metadata fields specified in Detection Guarantees.

Parameters
[in]mediaAn image or a video containing an unknown number of identities
[in]roleAn enumeration describing the intended function for the created template. Implementors are not required to have different types of templates for any/all of the roles specified but can if they choose.
[out]templatesA list of templates containing recognition information for all of the identities discovered in the media.
[out]tracksA list of metadata corresponding to the return templates.

janice_error janice_detect ( const janice_media &  media,
const size_t  min_face_size,
std::vector< janice_track > &  tracks 
)

Detect objects in a janice_media.

Each object is represented by a janice_track. Detected objects can then be used to create #janice_assocation and passed to janice_create_template.

Note
The number of attributes in a track can never exceed the number of frames in a janice_media instance. Still images should return tracks of length 1.

Minimum Face Size

A minimum size of faces to be detected may be specified by the caller of janice_detect. This size will be given as a pixel value and corresponds to the width of the face.

Detection Guarantees

The returned tracks will be ordered by decreasing janice_track::detection_confidence.

Each of the tracks will have values for at least janice_track::detection_confidence. Each of the janice_attributes within a track will have values for

  • janice_attributes::face_x
  • janice_attributes::face_y
  • janice_attributes::face_width
  • janice_attributes::face_height
  • janice_attributes::frame_number

Any attribute of the track or a janice_attributes within the track without a value will be set to NaN.

Parameters
[in]mediaMedia to detect objects in.
[in]min_face_sizeThe minimum width of detected faces that should be returned. The value is in pixels.
[out]tracksEmpty vector to be filled with detected objects.
Remarks
This function is thread_safe.

Ranked search for a template against a gallery.

template_ids and similarities are empty vectors to hold the return scores. The number of returns should be less than or equal to num_requested_returns, depending on the contents of the gallery.

The returned similarities may be normalized by the implementation based on the contents of the gallery. Therefore, similarity scores returned from searches against different galleries are not guaranteed to be comparable.

Parameters
[in]probeProbe to search for.
[in]galleryGallery to search against.
[in]num_requested_returnsThe desired number of returned results.
[out]template_idsEmpty vector to contain the janice_template_id of the top matching gallery templates.
[out]similaritiesEmpty vector to contain the similarity scores of the top matching templates.
Remarks
This function is thread_safe.
